# Gemini Learnings & Rules to Never Repeat

## The Modal Fix "Shitshow" - Core Failure Mode
Never again spend 10+ iterations blindly guessing pixel tweaks, padding reductions, or editing random components without verifying what is actually rendering.

### What Gemini Did Wrong:
1. **Blindly Guessing Files:** Trying to fix an overflow issue without confirming *which component* was actually triggering for the logged-out state (editing `ProfileRequiredModal` instead of `AuthChoiceModal`, or vice versa).
2. **Infinite Padding Tweaks:** Shrinking text, reducing margins by 2px, and adding `md:px-4` repeatedly instead of solving the structural CSS cause (container limits and overflow behavior).
3. **No Visual Verification:** Making assumptions about what the UI looked like instead of immediately opening the browser and checking the component's exact computed DOM (`getBoundingClientRect()`, viewport height, classes).

### What Claude Did Right (The 60-Second Fix):
1. **Identified the Root Cause Structurally:** Recognizing that when a modal's content *does* physically fit in the viewport, a scrollbar is only caused by the strict combination of `max-h-[100vh]` directly touching the viewport bounds combined with `overflow-y-auto`.
2. **Applied the Targeted Fix:** Changing `overflow-y-auto` to `overflow-hidden` on desktop (`md:overflow-hidden`) explicitly kills the scrollbar since the content already fits. Giving the modal `md:max-h-[90vh]` provides breathing room so it doesn't clip the screen edges.

### Golden Rules for UI Bugs Moving Forward:
- **STOP AND LOOK:** The absolute first step for *any* UI overflow or cut-off issue is to use the `browser_subagent` to visually confirm the bug and inspect the DOM. Do NOT guess CSS.
- **ISOLATE VIEWPORTS:** If a bug is on desktop, only touch `md:` or `lg:` classes. Leave mobile classes completely untouched.
- **AVOID MAGIC NUMBERS:** Fix layouts with structural Tailwind bounds (`overflow-hidden`, `max-h-[90vh]`, `flex-shrink-0`) rather than endlessly shaving off padding (`p-6` to `p-4`).
